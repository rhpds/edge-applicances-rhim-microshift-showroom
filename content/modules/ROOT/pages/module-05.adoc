== 5.1 Know: Connecting Solutions to Customer Problems

=== How Each Module Solved a Customer Challenge

Throughout this workshop, we've demonstrated how RHEL Image Mode and MicroShift solve the operational challenges that ManufacturingCo (and similar organizations) face when deploying edge appliances.

=== Module 1: Understanding Edge Challenges

**The Challenge:** Edge appliances are hard to manage due to configuration drift, risky updates, disconnected operations, resource constraints, and lifecycle management complexity.

**The Solution:** We introduced RHEL Image Mode as a way to treat the entire OS as an immutable, versioned image—just like a container. This provides consistency, reliability, simplicity, disconnected operation, and resource efficiency.

**The Outcome:** Participants understand why edge appliances are challenging and how RHEL Image Mode addresses these challenges at a fundamental level.

=== Module 2: Building Self-Contained Appliances

**The Challenge:** Disconnected operations—many edge devices have intermittent or no network connectivity, making traditional update and management approaches impractical.

**The Solution:** We built self-contained, "physically-bound" images that embed OS, MicroShift, and applications directly into the bootable image. This enables true offline operation.

**The Outcome:** Participants can build appliances that operate completely offline, eliminating the need for runtime registry access. This solves the disconnected operations challenge for ManufacturingCo's factory locations.

=== Module 3: Deploying to Edge Locations

**The Challenge:** Deployment complexity—how to get appliances to edge locations reliably and consistently, regardless of network connectivity.

**The Solution:** We deployed the appliance using ISO-based deployment to an isolated network, proving that embedded container images are used (not pulled from registries).

**The Outcome:** Participants verified that the appliance operates completely offline, with all images "already present on machine." This confirms that ManufacturingCo can deploy to any location regardless of connectivity.

=== Module 4: Safe Updates & Rollbacks

**The Challenge:** Risky updates—traditional update methods can break systems in the field, with 15% failure rate requiring on-site IT intervention and 2-4 hour recovery times.

**The Solution:** We demonstrated atomic updates with A/B boot model and instant rollback capability. Updates are staged safely, and if something goes wrong, the system rolls back instantly.

**The Outcome:** Participants experienced safe, atomic updates that can be instantly reverted. This eliminates the risky updates challenge for ManufacturingCo, reducing update failures from 15% to <1% and recovery time from 2-4 hours to instant rollback.

=== The Complete Solution

Together, these modules demonstrate a complete solution for edge appliance management:

* **Configuration Drift:** Solved by immutable, versioned images (Module 1, 2)
* **Risky Updates:** Solved by atomic updates with instant rollback (Module 4)
* **Disconnected Operations:** Solved by self-contained, physically-bound images (Module 2, 3)
* **Resource Constraints:** Solved by lightweight MicroShift (Module 1, 2)
* **Lifecycle Management Complexity:** Solved by container-based workflows (Module 1, 2, 4)
